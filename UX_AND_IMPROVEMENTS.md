# 🎨 UX/UI анализ и предложения по улучшению

## Анализ UX/UI

1. **Неочевидный UI выбора темы (Шаг 4):** Цветные круги (`.theme-round`) выглядят как декоративные элементы. У активной радиокнопки нет явного визуального акцента — обводки, галочки или изменения размера. Пользователю приходится всматриваться в маленькую точку сбоку (`.radio__square`), чтобы понять, какая тема выбрана. Это нарушает принцип видимости состояния системы (Nielsen's Heuristic #1).

2. **Отсутствие состояния загрузки (Loading state) для кнопки «Сгенерировать превью»:** Кнопка не переходит в состояние `:disabled` во время генерации кода и превью. Это позволяет пользователю отправлять множественные запросы подряд и не даёт визуальной обратной связи о том, что действие выполняется.

3. **Технически неточная инструкция по установке кода:** В тексте сказано: *"разместите код как можно ближе к началу страницы. Например, в пределах тегов `<head></head>` или `<body></body>`"*. Упоминание `<head>` вводит в заблуждение — iframe с виджетом должен вставляться в `<body>` в то место, где он должен отображаться. Размещение в `<head>` приведёт к тому, что виджет не будет виден пользователю.

## Предложения по улучшению (Improvements)

1. **Улучшить визуальную отдачу (Feedback) для радиокнопок выбора темы:** Добавить CSS-свойство `outline`, `box-shadow` или кастомную обводку (например, белое кольцо толщиной 2-3px) для активного `.theme-round`. Это стандартный паттерн для выбора цвета (используется в Figma, Material UI, Ant Design).

2. **Добавить состояние загрузки (Loading state) и debounce:** При нажатии на «Сгенерировать превью» кнопка должна переходить в состояние `:disabled` с отображением спиннера. В коде уже есть заготовка `data-events-widget-constructor-loading-value="Загрузка..."` — достаточно её реализовать в контроллере `events_widget_constructor_controller.js`.

3. **Скорректировать текст инструкции по установке:** Убрать упоминание тега `<head>` и заменить на более точную формулировку, например: *"Вставьте скопированный код в HTML-разметку вашего сайта в то место, где должен отображаться виджет (внутри тега `<body>`)"*.

4. **Исправить протокол для изображений промокодов:** Заменить абсолютные HTTP-ссылки `http://dev.3s.info/uploads/media/...` на протокол-относительные `//dev.3s.info/uploads/media/...` или HTTPS. Это устранит ошибки Mixed Content в консоли (см. BUG-03).

## ✅ Что реализовано хорошо (Positive Findings)

В ходе анализа кода были выявлены грамотно реализованные решения:

1. **Кнопка «Скопировать код»** (`#code-copy-button`) присутствует и имеет заготовленный текст подтверждения (`data-events-widget-constructor-copied-value="Скопировано"`).
2. **Валидация полей ввода размеров** реализована на уровне контроллера — поля очищаются от нецифровых символов, отрицательных значений и ограничены максимальными значениями (1020x720).
3. **Schema.org разметка** (BreadcrumbList) корректно внедрена для SEO.
4. **Использование Stimulus-контроллеров** (`events_widget_constructor_controller.js`, `copy_controller.js`) показывает современный подход к архитектуре фронтенда.
